Legal Language Elements (ToS, Privacy Policy)

All static documents legal documents like Privacy Policy, Terms of Service and Cardholder Agreement should be kept in form of static elements

Opt-in legal language will probably be stored in the database in the future to ensure control over different versions and

Naming convention for legal language elements

Proper element file name should contain three pieces: document title, promo type, specific promotion. If the document is intended to be used for multiple propmotions (generic document) or for many types of promotions, the appropiate part of the file name should be skipped

Examples:
Document Title Promotion Type Specific Promotion Actual Filename Comment
Terms and Conditions HMM PAH Generic terms_and_conditions_hmm.thtml Terms and Conditions for generic HMM site (can be shared accross all HMM PAH sites)
Terms and Conditions HMM PAH Skyfall terms_and_conditions_hmm_skyfall.thtml Terms and Conditions specific to HMM PAH Skyfall promotion (most likely used by skyfall controller)
Privacy Policy all all privacy_policy.thtml Privacy Policy shared accross all sites (different promo types)
Privacy Policy Virtual Visa Generic accountholder_agreement_virtualvisa.thtml Accountholder Agreement for generic Virtual Visa site (can be shared across all virtual visa sites)

Please do not use abbreviations in the first part of the name. Do not call "tos_hmm.thtml" but rather spell it out "terms_of_service_hmm.thtml"

HTML coding of elements

Each element should contain a div tag wrapping the whole content with proper id. Ids should be based on element title but CamelCased
